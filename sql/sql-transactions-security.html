Lesson 10

Transactions and Security

Transactions protect multi-step writes. Security practices protect data access and integrity.

Transaction Flow

BEGIN;

UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;

COMMIT;
-- Use ROLLBACK if an error occurs

Security Habits

  • Use parameterized queries in app code.
  • Grant least-privilege DB roles.
  • Audit destructive statements in production.
  • Back up before schema/data migrations.

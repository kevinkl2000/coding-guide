Troubleshooting

Common Git Issues and How To Fix Them

Use these step-by-step fixes for the errors teams hit most often in branch-based workflows.

Always Start Here

  • Run git status to check your exact branch and file state.
  • Run git branch to confirm where you are working.
  • If unsure, commit or stash before risky operations.
CONFLICT

Merge conflict during pull or merge

Git could not auto-combine edits in the same lines. Resolve manually, then complete the merge.

git status
# open conflicted files and edit sections between <<<<<<< and >>>>>>>
git add path/to/resolved-file
# repeat add for each resolved file
git commit
PUSH REJECTED

non-fast-forward push error

Remote branch moved ahead. Pull latest changes first, resolve conflicts if needed, then push again.

git fetch origin
git pull origin dev
# resolve any conflicts
git push origin dev
WRONG BRANCH

Committed work to main by mistake

Create a ticket branch from the mistaken commit, then safely undo the commit on main with revert.

git switch -c ticket/999-fix-branch-mistake
# push branch with your accidental commit
git push -u origin ticket/999-fix-branch-mistake

git switch main
git revert HEAD
git push origin main
UNCOMMITTED CHANGES

Cannot pull because local changes would be overwritten

Commit or stash first, then pull. Reapply stash after updating.

git status
git stash push -m "temp before pull"
git pull origin dev
git stash pop
REBASE STOPPED

Conflict during rebase

Resolve each conflict, stage fixed files, then continue rebase. Abort if you want to return to pre-rebase state.

git status
# resolve files
git add path/to/resolved-file
git rebase --continue
# if needed instead: git rebase --abort
DETACHED HEAD

You are not on a branch

Create a branch immediately if you made changes, so your work has a branch reference.

git status
git switch -c ticket/1000-save-detached-work
git push -u origin ticket/1000-save-detached-work

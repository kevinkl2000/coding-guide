Lesson 5

Merge and Rebase

Both merge and rebase combine work. Knowing when to use each keeps history understandable.

Merge vs Rebase

  • Use merge for explicit integration points.
  • Use rebase to replay local commits on top of updated branch history.
  • Never rebase shared branches without team agreement.

Command Example

git switch ticket/123-login-bug
git fetch origin
git rebase origin/dev
git switch dev
git merge --no-ff ticket/123-login-bug

Lesson 7

Production Ticket Workflow

This workflow matches a production team setup with main, dev, and ticket branches.

Branch Roles

  • main: production-ready code only.
  • dev: integration branch for upcoming release work.
  • ticket branches: focused work branches per task or bug.

Step-by-Step Flow

  1. Update main and dev locally.
  2. Create a ticket branch from main.
  3. Build and commit your ticket changes.
  4. Merge ticket branch into dev through pull request.
  5. After QA on dev, merge dev into main for release.

Concrete Commands

git switch main
git pull origin main
git switch dev
git pull origin dev

git switch main
git switch -c ticket/456-payment-timeout
# make changes
git add .
git commit -m "fix(payment): reduce timeout retries"
git push -u origin ticket/456-payment-timeout

# open PR: ticket/456-payment-timeout -> dev
# after merge to dev and testing passes:
# open PR: dev -> main

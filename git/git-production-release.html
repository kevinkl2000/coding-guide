Lesson 8

Production Releases

Stable release habits reduce risk and improve recovery when something goes wrong.

Release Checklist

  • Ensure dev is green and tested.
  • Merge dev into main via pull request.
  • Tag the release commit.
  • Deploy from tagged main commit.

Tag and Hotfix Commands

git switch main
git pull origin main
git tag -a v1.4.0 -m "Release 1.4.0"
git push origin v1.4.0

# hotfix path
git switch -c hotfix/1.4.1 main
# fix issue
git add .
git commit -m "fix(prod): patch checkout null edge case"
git push -u origin hotfix/1.4.1
# open PR: hotfix/1.4.1 -> main and then main -> dev

Lesson 5

State, Files, and Sessions

HTTP is stateless, so every request starts fresh unless you deliberately store state. This lesson explains how PHP applications remember users and organize multi-file projects.

Shared Files and Layouts

As your app grows, split repeated markup into partials and keep logic separate from templates when possible.

<?php
require __DIR__ . "/partials/header.php";
require __DIR__ . "/partials/navigation.php";
?>

Sessions

Sessions are ideal for login state, shopping carts, and temporary messages because the browser only holds a session identifier while the actual data stays on the server.

<?php
session_start();
$_SESSION["user_name"] = "Amina";
$_SESSION["role"] = "editor";

echo $_SESSION["user_name"] . " - " . $_SESSION["role"];
?>

Output

Amina - editor

Cookies

Cookies live in the browser and come back with later requests. Use them for light preferences such as theme, remember-me flags, or recently chosen settings, but never store sensitive plain data in them.

<?php
setcookie("theme", "dark", time() + 86400 * 30, "/");
?>

Guarding Private Pages

Protect dashboard or admin pages by checking session state before output.

<?php
session_start();

if (empty($_SESSION["user_id"])) {
    header("Location: /login.php");
    exit;
}

echo "Dashboard";
?>

Output

If not logged in:

Browser is redirected to /login.php

If logged in:

Dashboard

Uploads

Uploads are a common beginner trap because they combine forms, files, validation, and storage. Always validate file type, size, and target path before saving a file.

<?php
if (isset($_FILES["avatar"]) && $_FILES["avatar"]["error"] === UPLOAD_ERR_OK) {
    $allowedTypes = ["image/jpeg", "image/png"];
    $detectedType = mime_content_type($_FILES["avatar"]["tmp_name"]);

    if (in_array($detectedType, $allowedTypes, true)) {
        move_uploaded_file($_FILES["avatar"]["tmp_name"], __DIR__ . "/uploads/avatar.png");
    }
}
?>

Key Mental Model

  • Each request starts fresh unless you store state.
  • Sessions are server-side state tied to one browser session.
  • Cookies are browser-side values that return on later requests.
  • Private pages should check permissions before rendering HTML.

Lesson 4

Forms and User Input

Forms are where beginner PHP starts becoming real web development. You accept input, validate it, keep the user experience clear, and only trust clean data.

GET vs POST

Use GET when the request should be bookmarkable or visible in the URL, such as searching or filtering. Use POST when you are submitting data that changes the system or should stay out of the address bar.

Method Best Use Notes
GET Search, sorting, filtering Values appear in the URL and can be shared.
POST Registration, login, create and update actions Values are sent in the request body.

Basic Form Handling Pattern

A reliable beginner pattern is: detect the request, read values safely, validate them, then either show errors or continue processing.

<?php
if ($_SERVER["REQUEST_METHOD"] === "POST") {
    $name = trim($_POST["name"] ?? "");
    $email = trim($_POST["email"] ?? "");

    if ($name === "" || $email === "") {
        echo "Please complete all fields.";
    } else {
        echo "Thanks, " . htmlspecialchars($name);
    }
}
?>

Output

If fields are empty:

Please complete all fields.

If the form is valid:

Thanks, Amina

Collecting Validation Errors

Real forms often need to report more than one problem. Instead of failing on the first issue, collect errors in an array and show them together.

<?php
$errors = [];

if ($name === "") {
    $errors[] = "Name is required.";
}

if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
    $errors[] = "Email address is not valid.";
}

if (empty($errors)) {
    echo "Form is valid.";
}
?>

Output

Form is valid.

This pattern scales much better once you have five or ten fields on a form.

Sticky Forms

If validation fails, keep the values the user already typed. This improves usability and reduces frustration.

<form method="post" action="">
    <input type="text" name="name" value="<?= htmlspecialchars($name ?? "") ?>">
    <input type="email" name="email" value="<?= htmlspecialchars($email ?? "") ?>">
    <button type="submit">Send</button>
</form>

Output

Amina
amina@example.com
Send

Escaping Output

Validation checks the structure of data. Escaping protects the page when data is rendered. These are different jobs, and you need both.

<?php
echo htmlspecialchars($name, ENT_QUOTES, "UTF-8");
?>

File Upload Basics

Uploads require special form encoding and careful validation. Check the upload error, inspect the file type, and only then move the file into a controlled folder.

<?php
if (isset($_FILES["resume"]) && $_FILES["resume"]["error"] === UPLOAD_ERR_OK) {
    $tmpPath = $_FILES["resume"]["tmp_name"];
    $originalName = basename($_FILES["resume"]["name"]);
    move_uploaded_file($tmpPath, __DIR__ . "/uploads/" . $originalName);
}
?>

What Good Form Code Does

  • Checks the request method before processing.
  • Uses fallback values like $_POST["field"] ?? "".
  • Validates data on the server.
  • Escapes user data when rendering it into HTML.

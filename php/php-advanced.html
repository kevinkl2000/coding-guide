Lesson 8

APIs, Authentication, and Security

This final lesson introduces the habits that separate a demo from a real application: returning machine-readable responses, protecting user accounts, handling sessions correctly, and reducing attack surface.

Returning JSON for APIs

Not every PHP response should be HTML. API endpoints often return JSON so JavaScript apps, mobile apps, or external systems can consume data.

<?php
header("Content-Type: application/json");

echo json_encode([
    "status" => "success",
    "message" => "Guide loaded",
    "items" => ["syntax", "forms", "pdo"],
]);
?>

Output

{"status":"success","message":"Guide loaded","items":["syntax","forms","pdo"]}

Password Hashing

Passwords should never be encrypted with custom code or stored directly. Use the built-in password helpers so PHP chooses a safe hashing strategy for you.

<?php
$hash = password_hash($password, PASSWORD_DEFAULT);

if (password_verify($enteredPassword, $hash)) {
    echo "Password is correct.";
}
?>

Output

Password is correct.

Authentication Flow

  1. User submits email and password.
  2. Server finds the matching account by email.
  3. Server verifies the submitted password against the stored hash.
  4. Server stores the user ID in the session.
  5. Private pages check that session before rendering.

CSRF Protection

For sensitive form actions, include a token so the server can confirm the request came from your own application flow.

<?php
session_start();

if (empty($_SESSION["csrf_token"])) {
    $_SESSION["csrf_token"] = bin2hex(random_bytes(32));
}
?>

<input type="hidden" name="csrf_token" value="<?= $_SESSION["csrf_token"] ?>">

Output

<input type="hidden" name="csrf_token" value="4f8a21c2d20f7bdc1f84f0d17b4b7b61c4b0c3db63f7a8a2f3f4a14b7ea8d261">

Security Checklist

  • Escape HTML output to reduce XSS risk.
  • Use prepared statements to reduce SQL injection risk.
  • Regenerate session IDs after login.
  • Validate uploaded files before saving them.
  • Return proper HTTP status codes for API failures.

Production Mindset

Experienced PHP development is less about memorizing syntax and more about managing risk, reducing duplication, and making code predictable under real traffic and messy user input.

End-State Skills

  • You can read and return JSON from PHP.
  • You understand the basic login flow with sessions.
  • You know the core security protections every PHP app needs.
  • You can now move into frameworks with much better fundamentals.

Lesson 1

Getting Started With PHP

Begin here if you are new to server-side programming. This lesson explains what PHP is, how it runs, how PHP files differ from normal HTML files, and what a first useful script looks like.

What PHP Actually Does

PHP runs on the server, not in the browser. A browser asks for a page, the server executes the PHP file, and the browser only receives the final response. That response might be HTML, JSON, a redirect, or even a file download.

This matters because PHP is excellent at tasks the browser should not do alone: validating input, reading databases, managing sessions, and protecting sensitive logic.

The Request Lifecycle

  1. A browser requests a URL such as /profile.php.
  2. The web server sees that the file should be handled by PHP.
  3. PHP executes the code from top to bottom for that single request.
  4. The output is returned to the browser.
  5. On the next request, the process starts again.

That last point is important: PHP does not keep local variables alive between requests unless you store data in sessions, cookies, files, or databases.

Your First Script

The basic PHP opening tag is <?php. Use echo to send output back to the browser.

<?php
$language = "PHP";
echo "<h1>Welcome to $language</h1>";
?>

Output

Welcome to PHP

Mixing PHP With HTML

Many beginner PHP pages are mostly HTML with small PHP sections inside them. That is normal and often the easiest place to start.

<?php
$pageTitle = "Student Dashboard";
?>

<h2><?= $pageTitle ?></h2>
<p>Welcome to your first dynamic page.</p>

Output

Student Dashboard

Welcome to your first dynamic page.

The short echo tag <?= is a convenient way to print a value inside HTML.

Comments and Readability

Good PHP is readable PHP. Use comments for intent, not for obvious statements. Name files and variables so someone else can understand your code without reverse-engineering it.

<?php
// Store the display name for the signed-in user.
$displayName = "Amina Hassan";

# This single-line comment also works.

/*
   Multi-line comments are useful for explaining
   longer decisions or temporary notes.
*/
?>

Debugging Early

New developers often guess what a variable contains. That slows you down. Print values instead and look at them directly.

<?php
$user = [
    "name" => "Amina",
    "role" => "editor",
];

var_dump($user);
?>

Output

array(2) {
  ["name"]=> "Amina"
  ["role"]=> "editor"
}

By The End Of Lesson 1

  • You should understand that PHP executes on the server.
  • You should be comfortable reading a small PHP file with HTML inside it.
  • You should know how to print values and inspect them with var_dump().
  • You are now ready to learn PHP syntax in detail.

Lesson 3

Logic, Loops, and Functions

This is where PHP starts to feel like programming rather than markup. You make decisions, repeat work, and package logic into reusable pieces.

Conditionals

Use if, elseif, and else when your code should respond differently depending on data.

<?php
$score = 74;

if ($score >= 85) {
    echo "Excellent";
} elseif ($score >= 60) {
    echo "Pass";
} else {
    echo "Retry";
}
?>

Output

Pass

Loops

Loops help you render repeated data such as menus, task lists, and database results. In PHP, foreach is the most common loop you will use in web apps.

<?php
$topics = ["Routing", "Templates", "Forms"];

foreach ($topics as $topic) {
    echo "<li>" . htmlspecialchars($topic) . "</li>";
}
?>

Output

  • Routing
  • Templates
  • Forms

Functions With Parameters and Return Types

A function should do one clear job. Pass in what it needs and return the result. This keeps pages smaller and easier to test mentally.

<?php
function greetUser(string $name, bool $isAdmin): string
{
    if ($isAdmin) {
        return "Welcome back, admin " . $name;
    }

    return "Welcome back, " . $name;
}

echo greetUser("Amina", true);
?>

Output

Welcome back, admin Amina

Variable Scope

Variables declared inside functions stay inside those functions unless passed in or returned. This is good because it reduces accidental side effects.

<?php
$tax = 0.2;

function addTax(float $amount, float $tax): float
{
    return $amount + ($amount * $tax);
}

$total = addTax(100, $tax);
?>

Include and Require

As soon as a page contains repeated header, footer, or navigation markup, extract those pieces into separate files. This is the first step toward maintainable PHP projects.

<?php
require __DIR__ . "/partials/header.php";
include __DIR__ . "/partials/flash-message.php";
?>

Rendering Helper Example

A small helper function can make your views cleaner by hiding repetitive HTML generation.

<?php
function renderStatus(string $status): string
{
    return $status === "done" ? "Completed" : "In progress";
}

echo renderStatus("done");
?>

Output

Completed

What Changes Here

  • You stop hard-coding and start responding to data.
  • You render repeated items with loops instead of copy-paste.
  • You move repeated logic into functions and includes.
  • This prepares you for forms, sessions, and database results.

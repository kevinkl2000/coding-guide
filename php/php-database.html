Lesson 6

Databases With MySQL and PDO

This is where your application stops forgetting everything between requests. Databases let you persist users, tasks, orders, messages, and any other important records.

Why PDO

PDO gives you a consistent way to connect to databases, prepare statements, bind values, and handle errors. It is the standard baseline for modern PHP database work.

Creating a Connection

Keep database credentials centralized so the rest of the app can reuse the same connection setup.

<?php
$dsn = "mysql:host=localhost;dbname=school_app;charset=utf8mb4";
$username = "root";
$password = "";

try {
    $pdo = new PDO($dsn, $username, $password, [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    ]);
} catch (PDOException $exception) {
    exit("Database connection failed: " . $exception->getMessage());
}
?>

Reading Data Safely

Prepared statements protect your query structure by keeping user data separate from the SQL command.

<?php
$stmt = $pdo->prepare("SELECT * FROM users WHERE email = :email");
$stmt->execute(["email" => $email]);
$user = $stmt->fetch();
?>

The pattern is simple: prepare, execute, fetch.

Inserting Records

<?php
$sql = "INSERT INTO users (name, email, created_at) VALUES (:name, :email, NOW())";
$stmt = $pdo->prepare($sql);
$stmt->execute([
    "name" => $name,
    "email" => $email,
]);
?>

This is your create step in CRUD: create, read, update, delete.

Fetching Multiple Rows

<?php
$stmt = $pdo->query("SELECT id, title, status FROM tasks ORDER BY created_at DESC");
$tasks = $stmt->fetchAll();

foreach ($tasks as $task) {
    echo "<p>" . htmlspecialchars($task["title"]) . "</p>";
}
?>

Designing a Small Data Layer

Do not scatter raw SQL throughout every page. Even in small apps, create a simple repository or data access layer so database logic is easier to update later.

project/
|-- config/
|   |-- database.php
|-- public/
|   |-- index.php
|-- src/
|   |-- UserRepository.php
|-- templates/
|   |-- header.php

PDO Rules

  • Always enable exception mode.
  • Use prepared statements for any user-provided value.
  • Keep credentials out of public directories.
  • Escape database data when rendering it into HTML.

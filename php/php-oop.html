Lesson 4

Object-Oriented PHP

Object-oriented PHP helps you model real application concepts using classes, properties, and methods. It becomes especially useful when projects grow.

Basic Class Example

<?php
class User
{
    private string $name;
    private string $email;

    public function __construct(string $name, string $email)
    {
        $this->name = $name;
        $this->email = $email;
    }

    public function getDisplayName(): string
    {
        return strtoupper($this->name);
    }
}

$user = new User("Amina", "amina@example.com");
echo $user->getDisplayName();
?>

Output

AMINA

Core OOP Terms

  • Class: the blueprint for an object.
  • Object: one created instance of a class.
  • Property: data stored inside an object.
  • Method: behavior defined on a class.
  • Visibility: public, protected, or private access level.

Inheritance Example

<?php
class AdminUser extends User
{
    public function getRole(): string
    {
        return "Administrator";
    }
}
?>

Inheritance is helpful when a child class extends base behavior, but avoid deep class chains when composition would be simpler.

Repository Example

<?php
class UserRepository
{
    public function __construct(private PDO $pdo)
    {
    }

    public function findByEmail(string $email): array|false
    {
        $stmt = $this->pdo->prepare("SELECT * FROM users WHERE email = :email");
        $stmt->execute(["email" => $email]);

        return $stmt->fetch();
    }
}
?>

When OOP Helps

  • When you need reusable domain logic.
  • When classes can encapsulate database or API access.
  • When constructor injection makes dependencies explicit.
  • When you want cleaner separation between data and presentation.

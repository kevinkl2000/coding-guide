Lesson 7

Object-Oriented PHP

At this point you have enough procedural knowledge to appreciate why classes matter. OOP is not about making code look advanced; it is about organizing responsibility as your app grows.

Class Anatomy

A class groups related data and behavior. Use properties for state and methods for actions.

<?php
class User
{
    private string $name;
    private string $email;

    public function __construct(string $name, string $email)
    {
        $this->name = $name;
        $this->email = $email;
    }

    public function getDisplayName(): string
    {
        return strtoupper($this->name);
    }
}

$user = new User("Amina", "amina@example.com");
echo $user->getDisplayName();
?>

Output

AMINA

Why Encapsulation Matters

Keeping properties private prevents random parts of the application from mutating object state in uncontrolled ways. Public methods become the safe interface to that object.

  • private: only the class itself can access it.
  • protected: the class and its children can access it.
  • public: any code can access it.

Inheritance vs Composition

Inheritance can be useful, but many apps are clearer when objects collaborate instead of extending long class chains.

<?php
class AdminUser extends User
{
    public function getRole(): string
    {
        return "Administrator";
    }
}
?>

Use inheritance when the child is truly a specialized version of the parent. Use composition when an object merely needs another object to do a job.

Dependency Injection

Passing dependencies into constructors makes classes easier to understand and easier to replace in larger apps.

<?php
class UserRepository
{
    public function __construct(private PDO $pdo)
    {
    }
}
?>

Repository Example

A repository centralizes data access logic for one type of record. This keeps controllers or pages from becoming full of SQL.

<?php
class UserRepository
{
    public function __construct(private PDO $pdo)
    {
    }

    public function findByEmail(string $email): array|false
    {
        $stmt = $this->pdo->prepare("SELECT * FROM users WHERE email = :email");
        $stmt->execute(["email" => $email]);

        return $stmt->fetch();
    }
}
?>

When OOP Helps

  • When one concept needs both data and behavior.
  • When database logic should live outside page files.
  • When dependencies should be explicit and injectable.
  • When your project is large enough that organization matters more than raw speed of writing.
